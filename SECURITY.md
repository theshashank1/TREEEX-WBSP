## Reporting a Vulnerability

We take the security of TREEEX-WBSP seriously. If you have discovered a security vulnerability or potential threat in our codebase, we appreciate your help in disclosing it to us responsibly.

### How to Report

Please do not report security vulnerabilities through public GitHub issues. Instead, please report them via email:

1.  **Email**: security@treeex.example.com (Replace with actual email)
2.  **Subject**: `[SECURITY] - <Short Description of Vulnerability>`
3.  **Content**: Please include as much detail as possible:
    *   The type of issue (e.g., SQL Injection, XSS, RCE).
    *   Full steps to reproduce.
    *   Proof of concept code or screenshots.
    *   Impact of the vulnerability.

### Response Timeline

*   We will acknowledge your report within **48 hours**.
*   We will provide an estimated timeline for the fix within **5 business days**.
*   We will release a patch as soon as possible, depending on the severity.

### Bounty Program

Currently, we do not have an active bug bounty program, but we acknowledge and credit all security researchers in our [CHANGELOG](docs/CHANGELOG.md).
